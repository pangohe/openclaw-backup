## 🔍 逐步执行详情

### Step 1: Cron 触发

**时间**: 每5分钟（例如：14:00, 14:05, 14:10...）

```bash
# 自动执行的命令
python3 /root/.openclaw/workspace/scripts/token_watcher_v2.py --quiet
```

**参数说明**:
- `--quiet`: 静默模式，只记录到日志文件，不输出到控制台
- Cron 会自动传入这个参数

### Step 3: 评估会话状态

**阈值配置**:
```python
THRESHOLDS = {
    "warning": 60,      # 警告
    "compress": 70,    # 压缩 🚨 触发点
    "critical": 80,    # 严重
    "emergency": 90    # 紧急
}
```

**状态判断**:
```
Token%   状态      操作
─────────────────────────
< 60%    OK        ✅ 跳过
60-69%   WARNING   ⚠️ 记录
70-79%   COMPRESS  🗑️ 触发增强压缩
80-89%   CRITICAL  🗑️ 立即压缩
≥ 90%    EMERGENCY 🗑️ 紧急压缩
```

---

### Step 4: 增强压缩（≥70%时触发）

#### 备份原文件
```bash
# 自动创建带时间戳的备份
session.jsonl → session.jsonl.bak.20260301140500
```

#### 计算重要性评分
```python
score = 0.4 × position_score +    # 位置（近期消息更重要）
        0.3 × keyword_score +     # 关键词匹配（BM25）
        0.2 × length_score +      # 长度（偏好中等长度）
        0.1 × role_score          # 角色（user稍高）
```

#### 选择保留消息
1. 提取最近20条消息的关键词
2. 计算所有消息的评分
3. 选择评分最高的N条消息
4. **强制保留最近10条**（不管评分）
5. 按时间顺序输出

#### 写入压缩后的文件
```json
// session.jsonl（压缩后）
{"role": "user", "content": "最新消息1..."}
{"role": "assistant", "content": "回复1..."}
...
{"role": "user", "content": "最新消息10..."}
```

### 2. 自动备份

所有压缩操作都会先备份：

```bash
session.jsonl → session.jsonl.bak.YYYYMMDDHHMMSS
```

备份文件永远保留，可以随时恢复。

### 3. 降级模式

如果增强压缩失败，自动降级到原生压缩：

```python
try:
    # 尝试增强压缩
    result = compressor.compact_session(...)
except Exception as e:
    # 降级到原生压缩
    result = native_compact_session(...)
```
