| 组件 | 大小 | 占比 |
|------|------|------|

1. 每次对话都要读取并拼接这些文件

4. Nvidia GLM4.7 处理 76k tokens 需要 1-2 秒

- Gateway 进程重启，清理缓存
- 配置重载优化文件读取逻辑
- 新进程状态干净

**根本解决方案**：
- 压缩 MEMORY.md（保留核心记忆）
- 减少 skills 数量
- 优化工具描述

**技术洞察**：
- OpenClaw 的可观测性（sessions.json）让根因分析成为可能
- 数据驱动诊断 > 感觉猜测

## 📊 缺口交易策略详解

### 原理
```
正常情况：YES + NO = 1.0
缺口出现：YES + NO ≠ 1.0
例如：0.55 + 0.42 = 0.97 (3% 缺口)

套利策略：同时买入 YES 和 NO
- 购买 $275 YES @ 0.55
- 购买 $210 NO @ 0.42
- 总成本：$485

无论结果：
- YES 赢：$275 / 0.55 = $500
- NO 赢：$210 / 0.42 = $500
- 最少收益：$500 - $485 = $15 (3.1%)
```

### 预期收益
- **每日**：$120-150
- **每月**：$3,600-4,500
- **风险**：极低（YES+NO 缺口套利）
